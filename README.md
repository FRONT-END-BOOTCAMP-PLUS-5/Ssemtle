## ✨서비스 소개

반응형 웹 앱 입니다.

*선생님*
1. 학생 일괄등록으로 자신이 관리하는 학생들의 회원가입을 한 번에 시킬 수 있습니다.

2. 관리하는 학생들의 마이페이지를 통해 학습 상태를 그래프로 한 눈에 파악하거나, <br />
구체적으로 어떤 문제를 틀렸는지 확인할 수 있습니다.

3. 단원평가 문제 생성을 통해, 쪽지 시험을 클릭 몇 번으로 생성할 수 있습니다.

*학생*
1. 선택한 단원의 무한한 문제풀이가 가능합니다.
2. 문제풀이의 해답 및 해설을 확인하여 자가 학습이 가능합니다.
3. 자신이 풀었던 모든 문제풀이 내역을 확인할 수 있습니다.
4. 오답노트 기능을 이용하여 틀린 문제만 확인 및 다시 풀어볼 수 있습니다.
5. 마이페이지에서 자신의 학습 성취도를 그래프를 통해 한 눈에 확인할 수 있습니다.

*관리자*
1. 선생님 회원가입 승인,거절이 가능합니다.
2. 학습 단원을 추가,삭제,수정할 수 있습니다.
(관리자는 일반 회원은 가입 및 이용할 수 없습니다.)

### 기획 의도, 목적

- 기획 배경
코로나 팬데믹 이후 사회적으로 기초학력 저하 문제가 대두되고 있습니다. <br />
기초학력 저하는 단순히 한 개인의 성적 하락에 국한되는 것이 아니라 학생의 미래와 사회의 지속 가능성과 직결된 문제 입니다.

- 타겟층
중1~3학년의 수학 기초학력 학생과 그들의 지도를 쉽게 하기 위한 선생님들을 위한 웹 앱 입니다. <br />
*기초학력 학생은 기초학력 뿐만 아니라, 해당 단원을 처음 배우는 학생들을 포함 (기초학력의 실력) <br />
*타겟층이 선생님인 이유는, 시장 조사 결과 기초학력 학생들이 자체적으로 앱을 이용하는 경우는 거의 없기 때문에 선생님을 타겟으로 삼았습니다.<br />

 - 그래서 앱은 어떤 방향을 추구하는가? <br />
반복 학습을 통해 기초학력 탈피하도록 도와주는것이 목표 입니다.

## ⚙️페이지 별 기능 설명

### 문제풀기

[PC] <br />
![단원평가-desk](https://github.com/user-attachments/assets/e4543683-9654-4edd-b530-3d76e90f1c8f) <br />
[Mobile] <br />
![단원평가-mobile](https://github.com/user-attachments/assets/27c331b1-c7db-4a81-b5bd-8225f3b8da6c) <br />

문제풀기는 AI를 통한 무한한 문제풀이를 제공합니다. <br/>
풀고싶은 단원 카테고리 카드를 클릭하면, 미리 작성해 놓은 프롬프트를 통해 AI에게 문제 요청 및 응답을 받은 뒤 화면에 렌더링 하는 형태입니다. <br />
한 번의 API 통신에 10개의 문제(데이터)를 전달받으나, 바로 DB에 저장하지 않고, 유저가 문제를 제출한 경우에만 DB에 저장이 되도록 기획하였습니다. <br />

영상의 흐름은 다음과 같습니다. <br />
카테고리 선택 -> 쿼리파라미터로 단원 식별 -> 문제 배치 생성(큐) -> 제출 시 서버 채점/DB저장 -> 다음 문제 전환 <br />

- 문제풀이 페이지 최초 진입 시 DB의 unit 목록을 조회하여 단원 목록을 렌더링합니다. <br />
- 검색 입력 시 클라이언트에서 목록을 필터링하여 렌더링 합니다.<br />
- 단원 목록 카드 선택 시 문제풀이 페이지로 이동합니다.<br />
- URL 쿼리에서 unitId, unitName을 읽어 현재 단원을 결정 및 <br />
DB의 unit 목록에서 해당 unitId의 학습 도움용 영상 URL을 렌더링합니다.<br />
- 따로 만든 가상 숫자패드/연산자 버튼을 제공합니다.<br />
- 키보드 입력 감지 비활성화 등을 사용자에게 알려주는 등 사용자 실수를 줄이는 처리도 진행하였습니다.<br />
- 정답을 제출하면 서버사이드에서 정답을 검증하고 채점합니다.<br />
- 정,오답과 관계없이 제출된 데이터는 DB의 solve 테이블에 레코드로 저장됩니다.<br />
(해당 부분은 학습 이력/ 오답노트/ 통계에 활용됩니다.)<br />
- 문제는 기본적으로 최초 생성시 10개의 데이터가 생성되며, 배치 내 다음 인덱스가 있으면 즉시 전환됩니다.<br />
만약, 문제가 없다면 API 통신을 다시 진행합니다.<br />
- 생성된 문제는 DB에 저장되지 않으며, 사용자가 제출 버튼을 클릭한 문제에 대해서만 DB의 question 테이블에 문제가 저장됩니다.

###  문제집

[PC] <br />
![문제집-desk](https://github.com/user-attachments/assets/aa796124-ced5-4bee-9b9a-afed3a74bcd4)<br />
[Mobile] <br />
![문제집-mobile](https://github.com/user-attachments/assets/84b0b7c8-6c92-4136-a576-3d7c4e180d91) <br />

문제집은 내가 지금까지 풀이했던 내용을 모두 담은 말 그대로의 문제집 입니다. <br />
하루를 기준으로, 각 카테고리별로 나누어서 분류하였습니다. <br />

흐름은 다음과 같습니다. <br />
각 문제집 카드 클릭 -> 에러노트로 이동 -> 쿼리 파라미터 기준으로 오답 조회 -> 재풀이 <br />

- 최초 페이지 접근 시, 로그인 된 ID를 확인하여 DB의 solve 테이블에서 해당 ID가 풀이했던 데이터를 무한 스크롤로 불러옵니다. <br />

- 단원 필터와 날짜 정렬을 선택하면 URL 쿼리로 반영하고, 목록을 재조회 합니다. <br />

- 문제 카드 클릭 시 에러노트로 이동합니다.(쿼리 파라미터로 파싱) <br />

- 쿼리에 있는 데이터를 기반으로 서버에서 해당하는 내용의 오답 항목을 조회하여 화면에 렌더링합니다. <br />

- 각 오답 문제는 가상 키보드를 통해 즉시 재풀이가 가능합니다. <br />
<br/>

###  단원평가

[PC] <br />
![단원평가-desk](https://github.com/user-attachments/assets/c2944ad7-3c34-452d-b389-7f60a641d491) <br/>
[Mobile] <br />
![단원평가-mobile](https://github.com/user-attachments/assets/1ff7e568-2ad4-424a-bc47-9cd952b05f4d) <br />

단원평가는 선생님 계정으로 단원평가를 생성하면 만들어지는 코드를 통해 <br />
학생들이 단원평가 시험을 볼 수 있도록 하는 기능입니다. <br />

흐름은 다음과 같습니다. <br />
코드 검증 -> 문제 로드(진입 기록) -> 제한시간/진행 관리 -> 일괄 제출(저장) -> 결과페이지 요약 <br />

- 유저가 단원평가 코드를 입력하면 형식 검증을 수행합니다 <br />
- 올바르지 않거나, 이미 응시한 시험인 경우는 토글 메세지로 안내를 합니다. <br />
- 유효한 코드인 경우 서버에서 해당 코드의 문제 목록을 가져옵니다.  <br />
* 이 때 최초 진입 시도 기록을 생성합니다. <br />
* 이 기록을 통해 사용자가 응시한 시험에 대해 다시 재응시를 할 수 없도록 기획하였습니다. <br />
- 유저는 각 문항에 답안을 입력하며, 이전/다음 이동으로 문제를 수정할 수 있습니다. <br />
- 미제출하거나 응시 시간이 다 된 경우 제출하지 못한 답은 미응시로 오답 처리 됩니다. <br />
- 결과 페이지에서 응시 결과를 기반으로 총 점수를 바로 알려줍니다.(해설도 같이 제공) <br />
- 단원평가 페이지에서 지금까지 응시했던 모든 단원평가를 확인해볼 수 있습니다. <br />
